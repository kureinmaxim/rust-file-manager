# GUIDE: Rust с нуля — как начать писать правильно

Практический путеводитель для начинающего: инструменты, базовая грамматика, главные правила языка и привычки, с которых стоит начинать современную разработку. Примеры по возможности привязаны к коду этого репозитория — всё можно открыть и потрогать.

Дополняет [GUIDE_xtask.md](GUIDE_xtask.md), где построчно разобран небольшой реальный инструмент.

---

## 1. Инструменты: что поставить в первый день

```bash
# Единственный правильный способ установки — rustup (менеджер версий Rust)
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh

rustc --version    # компилятор
cargo --version    # сборщик, менеджер пакетов, запускатель тестов — всё в одном
```

Сразу же — три вещи, без которых не стоит писать ни строчки:

| Инструмент | Команда | Зачем |
|---|---|---|
| **rust-analyzer** | расширение для VS Code / RustRover | подсказки типов, ошибки прямо в редакторе, автодополнение |
| **rustfmt** | `cargo fmt` | автоформатирование; споры о стиле в Rust закрыты — стиль один на всех |
| **clippy** | `cargo clippy` | линтер: находит не-идиоматичный и подозрительный код и показывает, как правильно |

> **Привычка №1 современной разработки:** `cargo fmt` и `cargo clippy` гоняются с первого дня, а не «потом». Clippy — это бесплатный наставник: половина его подсказок — мини-уроки идиоматичного Rust.

## 2. Первый проект

```bash
cargo new my-app        # создаёт проект: Cargo.toml + src/main.rs + git-репозиторий
cd my-app
cargo run               # собрать и запустить
cargo check             # быстрая проверка «компилируется ли» без сборки бинарника
cargo test              # запустить тесты
```

Структура минимальна:

```
my-app/
  Cargo.toml      # манифест: имя, версия, зависимости (аналог package.json / pyproject.toml)
  src/main.rs     # точка входа: fn main()
```

`Cargo.lock` фиксирует точные версии зависимостей — для приложений его **коммитят** в git (воспроизводимая сборка), для библиотек — нет.

## 3. Переменные и базовые типы

### Неизменяемость по умолчанию

```rust
let x = 5;          // неизменяемая — попытка x = 6 не скомпилируется
let mut y = 5;      // изменяемая — mut пишется явно
y += 1;
```

Это первое «правило» языка: **всё константно, пока вы явно не попросили иначе**. В коде сразу видно, какие данные могут меняться, — читать такой код намного легче.

### Скалярные типы

```rust
let a: i32 = -42;        // целые со знаком: i8, i16, i32 (по умолчанию), i64, i128, isize
let b: u64 = 42;         // без знака: u8 ... usize (usize — для индексов и размеров)
let c: f64 = 3.14;       // дробные: f32, f64 (по умолчанию)
let d: bool = true;
let e: char = 'я';       // 4 байта, любой Unicode-символ
```

Типы почти всегда можно не писать — компилятор **выводит** их сам (`let a = -42;`). Пишут их на границах: в сигнатурах функций и полях структур — там это документация.

### Строки: `String` и `&str` — первое место, где все спотыкаются

```rust
let s: &str = "привет";            // срез строки: ссылка на текст, владельцем не является
let owned: String = s.to_string(); // String: владеющая, растущая строка (буфер в куче)
let back: &str = &owned;           // из String всегда можно дёшево получить &str
```

Практическое правило: **в параметрах функций принимайте `&str`** (подойдёт и литерал, и `String`), **храните в структурах `String`** (структура должна владеть своими данными). Та же пара есть у путей (`&Path` / `PathBuf`) и массивов (`&[T]` / `Vec<T>`) — выучив принцип один раз, вы понимаете все три.

### Коллекции

```rust
let v: Vec<i32> = vec![1, 2, 3];               // вектор — главный «список»
let pair: (i32, &str) = (1, "один");           // кортеж; pair.0, pair.1
let arr: [u8; 3] = [1, 2, 3];                  // массив фиксированной длины

use std::collections::HashMap;
let mut map: HashMap<String, u32> = HashMap::new();
map.insert("a".into(), 1);
```

## 4. Функции, выражения и управление

### Почти всё — выражение

```rust
fn add(a: i32, b: i32) -> i32 {
    a + b                    // без точки с запятой = это возвращаемое значение
}

let status = if count > 0 { "есть" } else { "пусто" };   // if — тоже выражение
```

Точка с запятой превращает выражение в оператор (значение отбрасывается). Забытая/лишняя `;` — типичная ошибка первой недели; компилятор про неё скажет прямо.

### Циклы

```rust
for x in 0..10 { ... }              // диапазон: 0..10 (без 10), 0..=10 (с 10)
for item in &collection { ... }     // по ссылкам на элементы (не забирая владение!)
while condition { ... }
loop { break; }                     // бесконечный цикл с явным выходом
```

### `match` — главный инструмент ветвления

```rust
match code {
    200 => println!("ok"),
    301 | 302 => println!("redirect"),
    400..=499 => println!("client error"),
    other => println!("что-то ещё: {other}"),
}
```

`match` **исчерпывающий**: компилятор требует обработать все случаи. Добавили новый вариант в enum — все `match` по нему перестанут компилироваться, пока вы их не дополните. Это не придирки, а защита: невозможно «забыть обработать случай».

## 5. Владение и заимствование — сердце языка

Это то, что отличает Rust от всего остального. Три правила:

1. У каждого значения есть ровно **один владелец**.
2. Когда владелец выходит из области видимости, значение **освобождается** (никакого сборщика мусора — и никаких ручных `free`).
3. Передача значения = **перемещение** владения (move), после которого старая переменная недействительна.

```rust
let s1 = String::from("hello");
let s2 = s1;                  // владение ПЕРЕМЕСТИЛОСЬ в s2
// println!("{s1}");          // ошибка компиляции: s1 больше не владеет значением
```

Чтобы пользоваться значением, не забирая его, существует **заимствование** — ссылки:

```rust
fn len(s: &String) -> usize { s.len() }    // & — одолжить на чтение

let s = String::from("hello");
let n = len(&s);          // одолжили
println!("{s}");          // s по-прежнему наша
```

Правило ссылок: **либо сколько угодно читающих `&T`, либо ровно одна пишущая `&mut T`** — одновременно и то и другое нельзя. Компилятор проверяет это статически. Именно отсюда главные обещания Rust: нет висячих указателей, нет гонок данных, нет «использования после освобождения» — целые классы ошибок C/C++ просто не компилируются.

> **Как это переживать в первый месяц:** когда borrow checker ругается — не воюйте с ним, читайте сообщение до конца (там почти всегда готовый совет) и помните, что `.clone()` — легальный выход, пока вы учитесь. Сначала рабочий код, оптимальные заимствования — потом.

## 6. Структуры, перечисления и методы

```rust
struct User {                       // данные
    name: String,
    active: bool,
}

impl User {                         // поведение — отдельно
    fn new(name: &str) -> Self {                 // конструктор по соглашению
        Self { name: name.to_string(), active: true }
    }
    fn deactivate(&mut self) {                   // метод, изменяющий данные
        self.active = false;
    }
}
```

`enum` в Rust мощнее, чем в других языках: каждый вариант может **нести данные**.

```rust
enum Shape {
    Circle { radius: f64 },
    Rect { w: f64, h: f64 },
}

fn area(s: &Shape) -> f64 {
    match s {
        Shape::Circle { radius } => 3.1415 * radius * radius,
        Shape::Rect { w, h } => w * h,
    }
}
```

`enum` + `match` — это способ моделировать «значение бывает таким ИЛИ таким» так, чтобы компилятор следил за полнотой обработки. На этом построена вся обработка ошибок.

## 7. `Option`, `Result` и `?` — обработка ошибок без исключений

В Rust **нет null и нет исключений**. Вместо них два enum из стандартной библиотеки:

```rust
enum Option<T> { Some(T), None }              // «значение может отсутствовать»
enum Result<T, E> { Ok(T), Err(E) }           // «операция может провалиться»
```

Компилятор не даст использовать `Option<String>` как `String` — сначала придётся честно разобрать оба случая:

```rust
match map.get("key") {
    Some(value) => println!("{value}"),
    None => println!("нет такого ключа"),
}
```

Для цепочек ошибок есть оператор `?` — «если ошибка, верни её вызывающему»:

```rust
fn read_config() -> Result<String, std::io::Error> {
    let text = std::fs::read_to_string("config.json")?;   // ошибка? — выходим с ней
    Ok(text)
}
```

Полезные методы, которые встречаются постоянно: `.unwrap_or(default)`, `.unwrap_or_else(|| ...)`, `.map(...)`, `.ok_or(...)`. А вот `.unwrap()` (паника при `None`/`Err`) в боевом коде — запах; он допустим в тестах и прототипах.

> Живой пример из этого репозитория: [`src/config.rs`](src/config.rs) — каждая переменная окружения читается через `Result`, и для каждой решено, что делать при отсутствии: default, понятная ошибка или предупреждение.

## 8. Итераторы и замыкания

Идиоматичный Rust обрабатывает коллекции цепочками, а не индексами:

```rust
let evens: Vec<i32> = (1..=10)
    .filter(|n| n % 2 == 0)     // |n| ... — замыкание (безымянная функция)
    .map(|n| n * n)
    .collect();                 // собрать результат в коллекцию
```

Это не «синтаксический сахар ценой скорости»: итераторы компилируются в тот же машинный код, что и ручной цикл (принцип *zero-cost abstractions* — абстракции бесплатны). Часто употребляемые методы: `filter`, `map`, `find`, `find_map`, `any`, `all`, `sum`, `collect`, `enumerate`, `zip`.

## 9. Трейты — интерфейсы по-растовски (минимум для старта)

```rust
trait Greet {
    fn greet(&self) -> String;
}

impl Greet for User {
    fn greet(&self) -> String { format!("Привет, {}!", self.name) }
}
```

Трейт описывает поведение; типы его реализуют. Самые ходовые трейты стандартной библиотеки вы получаете бесплатно через `#[derive(...)]`:

```rust
#[derive(Debug, Clone, PartialEq)]    // печать {:?}, копирование .clone(), сравнение ==
struct Point { x: i32, y: i32 }
```

В экосистеме то же самое делают `serde::Serialize`/`Deserialize` — «бесплатная» сериализация структур в JSON и обратно (см. структуры ответов в [`src/files.rs`](src/files.rs)).

## 10. Как правильно начинать современный проект — чек-лист

Привычки, которые дешевле завести с первого коммита, чем прикручивать потом.

### Качество кода — автоматически, не силой воли

```bash
cargo fmt                      # перед каждым коммитом
cargo clippy --all-targets     # и читать, что он говорит
cargo test                     # тесты лежат рядом с кодом
```

Тест в Rust — это четыре строчки в том же файле, что и код ([пример](src/categories.rs)):

```rust
#[cfg(test)]
mod tests {
    use super::*;

    #[test]
    fn it_works() { assert_eq!(add(2, 2), 4); }
}
```

Порог входа в тестирование настолько низкий, что писать первый тест нужно в первый же день.

### Зависимости — осознанно

- Добавляйте через `cargo add serde` (сам найдёт свежую версию).
- Перед добавлением спросите себя: это экономит сотни строк (веб-фреймворк, сериализация) или десять (можно написать самому)? У xtask этого проекта **ноль** зависимостей — осознанно.
- Версии в Rust следуют semver; `serde = "1"` означает «любая совместимая 1.x».

### Ошибки — продумать политику сразу

- Библиотечный код возвращает `Result`, **никогда не паникует**.
- Приложение наверху решает, что делать: показать пользователю, залогировать, упасть с кодом выхода.
- `unwrap()`/`expect()` — только там, где сбой действительно невозможен или это инструмент разработчика (как в xtask).

### Конфигурация и секреты

- Конфиг — из переменных окружения / `.env`, никогда не хардкодом (см. [`src/config.rs`](src/config.rs)).
- Секреты не попадают в git: `.env` в `.gitignore` с первого коммита, в репозитории — только `.env.example` с пустыми полями.

### Логи — `tracing`, а не `println!`

```rust
tracing::info!(user = %name, "login successful");   // структурированное событие
```

`println!` — для вывода программы, логи — для наблюдения за ней. Уровни (`error/warn/info/debug`) фильтруются переменной `RUST_LOG` без перекомпиляции.

### Документация — в коде

```rust
/// Считает площадь фигуры в квадратных метрах.
///
/// Возвращает 0.0 для вырожденных фигур.
fn area(s: &Shape) -> f64 { ... }
```

Комментарии `///` — это документация: `cargo doc --open` собирает из них сайт, rust-analyzer показывает их при наведении. Примеры кода внутри `///` даже **запускаются как тесты** (`cargo test`).

### Версионирование и релизы

Версия — в одном месте (`Cargo.toml`), меняется инструментом, а не руками; derived-файлы синхронизируются автоматически. Как это устроено здесь — [VERSION_MANAGEMENT.md](VERSION_MANAGEMENT.md) и [GUIDE_xtask.md](GUIDE_xtask.md).

### CI с первого дня (когда проект на GitHub)

Минимальный `.github/workflows/ci.yml` гоняет `fmt --check`, `clippy` и `test` на каждый push — и ни один сломанный коммит не пройдёт незамеченным. Это 20 строк YAML, добавляется за 10 минут.

## 11. Типичные грабли новичка (и как их обходить)

| Грабли | Что делать |
|---|---|
| «Воюю с borrow checker» | Читать ошибку до конца — совет обычно внутри. На первых порах `.clone()` — нормально. |
| `String` vs `&str` путаница | Параметры — `&str`, поля структур — `String`. |
| `unwrap()` повсюду | В прототипе — ладно; перед «настоящим» кодом заменить на `?` или явную обработку. |
| Индексы вместо итераторов | `for (i, x) in v.iter().enumerate()` вместо `for i in 0..v.len()`. |
| Игнорирование clippy | Он почти всегда прав. Каждое предупреждение — мини-урок. |
| Попытка выучить всё до первого проекта | Наоборот: маленький реальный проект (CLI-утилита!) с первой недели. Язык учится пальцами. |

## 12. Маршрут обучения

1. **[The Rust Book](https://doc.rust-lang.org/book/)** — официальный учебник; главы 1–9 покрывают всё из этого гайда.
2. **[Rustlings](https://github.com/rust-lang/rustlings)** — десятки крошечных упражнений «почини код, чтобы скомпилировалось»; идеально параллельно книге.
3. **[Rust by Example](https://doc.rust-lang.org/rust-by-example/)** — справочник «как пишется X» на готовых примерах.
4. **Этот репозиторий** — маленький, но настоящий проект: `src/` (~600 строк) можно прочитать целиком, [GUIDE_xtask.md](GUIDE_xtask.md) разбирает инструмент построчно.
5. Когда освоитесь: официальная документация любого crate живёт на [docs.rs](https://docs.rs) — например, [docs.rs/actix-web](https://docs.rs/actix-web).

Главный совет: **компилятор Rust — не противник, а парный программист**. Его сообщения об ошибках — лучшие в индустрии: с контекстом, объяснением и предложением исправления. Читайте их целиком — это и есть основной учебник.
